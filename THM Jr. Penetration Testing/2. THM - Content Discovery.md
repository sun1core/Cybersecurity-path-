*Note: Learn the various ways of discovering hidden or private content on a webserver that could lead to new vulnerabilities.*

Actual Meaning Content Discovery:
---> Finding hidden, unlinked, or non-obvious parts of a web application that are not directly visible on the website's main pages.

---> Actively searching for hidden or forgotten parts of a website that were not meant to be easily accessible or are not linked publicly.

###### Example
- Admin panels (e.g., `/admin`)
- Old or backup files (e.g., `/backup.zip`, `/index_old.html`)
- Hidden directories (e.g., `/internal/`, `/staff/`)
- API endpoints (e.g., `/api/v1/users`)
- Configuration files (e.g., `.env`, `config.json`)
- Development or test pages (e.g., `/dev`, `/staging`)




#### Manual Discovery - Robots.txt
- tells search engines which pages or folders they should not crawl or show in search result.
- reveal hidden paths like admin pages, backups or private directories.
---> Like ---> /staff-portal/ ---> private login area for employees to manage internal website or company operations.
